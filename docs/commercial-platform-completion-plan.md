# 禾语 AI 商业级平台完整建设架构

## 目标

将 README“接下来建设”的八个方向全部推进为真实、可运行、可测试的产品能力，同时保持：

- 农户视角的一条连续内容经营链路，而不是分散工具箱；
- 本地零成本 Mock 可演示，接入国产模型后可获得真实生成效果；
- 外部平台未授权时明确显示“未连接”，不虚构实时热点、自动发布或效果承诺；
- 简体中文、香港繁体中文、英文三语；
- 多组织隔离、角色权限、审计和可恢复运行。

  ├─ 热点信号服务
  ├─ 混合 RAG 服务
  ├─ 模型网关与质量评测
  ├─ 视频理解任务
  ├─ 发布任务状态机
  └─ 导入、审计、监控与恢复
          │
          ▼
适配器层
  ├─ 国产 OpenAI-compatible 模型
  ├─ Embedding / Rerank / STT / 多模态模型
  ├─ 合规热点数据源
  ├─ 第三方发布平台

长耗时任务统一进入可恢复任务队列；首版使用数据库任务表和进程内 Worker，避免强制安装
Redis、Docker 或本地大模型，后续可无缝替换为独立队列服务。

## “有热点、有热度潜力”的脚本生成链路

平台不能承诺爆款，也不能把一个热搜词硬塞进农产品文案。正确链路是：

1. **采集信号**：从已授权或允许使用的数据源、人工导入、节气日历和团队历史数据获取候选选题。
2. **保留证据**：每条信号保存来源、抓取时间、地区、平台、原始标题、指标和使用权限。
3. **先匹配再生成**：按产品相关性、目标人群、经营目标、平台表达方式、时效性和风险进行评分。
4. **拒绝弱匹配**：相关性不足时明确建议“不追这个热点”，转向节气、场景或用户痛点选题。
5. **生成差异路线**：同一选题至少给出吸睛实用、人物故事、轻松反差三条路线，农户可选择。
6. **形成完整拍摄包**：输出开头钩子、口播、分镜、字幕、音乐情绪、拍摄清单、标题、封面和评论区互动。
7. **质量门禁**：检查事实依据、前三秒钩子、平台适配、可拍性、重复度、敏感表达和 CTA。
8. **数据回流**：发布后导入播放、完播、互动、咨询和成交数据，用于下一轮选题和改稿。

热点匹配评分首版采用可解释的加权模型：

```text
总分 =
  产品与场景相关性 × 30%
  目标人群相关性 × 20%
  发布平台适配度 × 15%
  时效性 × 15%
  可拍摄性 × 10%
  证据可信度 × 10%
  - 合规与品牌风险扣分
```

模型只负责理解和创作，来源、时间、权限、分数和最终选择由确定性业务逻辑管理。

## 八个方向的实施架构

### 1. 知识库混合 RAG

- 文档解析后按标题、段落和语义边界切分，保留页码、来源和修订链。
- SQLite 本地模式使用 FTS5 + 小规模向量精确检索。
- PostgreSQL 商业模式使用全文检索 + pgvector。
- 关键词与向量结果使用 RRF 融合，再通过可配置 Rerank 适配器精排。
- 生成结果保留引用片段、来源和版本；建立检索命中与引用正确性评测。

### 2. 热点辅助

- 建立 `TrendProvider` 适配器和 `TopicSignal` 标准数据结构。
- 支持人工录入、CSV 导入、节气/农时数据和授权在线来源。
- 后台同步任务记录抓取时间、游标、错误、重试和来源条款。
- 选题页面先展示“为什么适合 / 为什么不适合”，再进入脚本生成。

### 3. 模型质量评测

- 建立脱敏农产品场景题集，覆盖多品类、多平台和三种语言。
- 同时评估结构合法性、事实引用、营销表达、平台适配、可拍性、三语质量、延迟和成本。
- 规则评分与模型评分分离；模型评分必须保存评分模型和提示词版本。
- 管理员可比较不同 Provider / 模型并查看回归趋势。

### 4. 运营数据回流

- 支持 CSV 和 Excel 的字段识别、预览、映射、校验、去重和正式导入。
- 数据绑定发布记录、内容版本和产品，避免成为无上下文数字。
- 生成“发生了什么—可能原因—下一步实验—可直接生成的新草稿”。
- 平台 API 连接通过独立适配器实现，未授权时不显示为已连接。

### 5. 自动视频理解

- 视频上传进入异步任务，计算文件摘要并保存安全元数据。
- 媒体探测、音频抽取、语音转写、镜头切分、字幕切分和多模态理解均为可替换适配器。
- 无外部模型时仍可完成文件检查、时长/画幅分析、人工字幕导入和演示降级。
- 诊断结果落到可编辑改进 Brief，并可一键生成下一版本脚本。

### 6. 发布协作

- 第一层提供平台专用复制包、字幕文件、拍摄清单和导出文件。
- 第二层建立定时发布状态机：草稿、待授权、已排期、发布中、成功、失败、重试中、已取消。
- Provider 保存外部任务 ID、回执、错误类别和幂等键。
- 只有获得合法授权的平台适配器才能执行真实发布。

### 7. 模型设置界面

- 管理员可配置文本生成、Embedding、Rerank、STT 和多模态 Provider。
- 密钥加密后只保存在服务端，API 永不返回明文。
- 提供连接测试、模型列表校验、超时、预算、降级策略和最近健康状态。
- 本地默认仍为 Mock，不要求 Ollama。

### 8. 正式商业运行

- 增加结构化日志、健康指标、任务积压、Provider 延迟/错误率和审计查询。
- 实现隐私政策、数据导出/删除、保留期限和清理任务。
- 编写备份、恢复和演练脚本，并在 CI 中验证空库迁移和备份恢复。
- 增加依赖、安全配置和上传边界检查；形成可重复的发布验收证据。

## 实施顺序

### 阶段 A：共享基础

1. Provider 配置与服务端密钥管理
2. 通用后台任务、状态、重试、幂等和审计
3. 三语导航、管理员页面和下一步引导骨架

### 阶段 B：内容生产核心

4. 混合 RAG
5. 热点信号与匹配
6. 三路线脚本生成和质量门禁
7. 模型质量评测

### 阶段 C：发布与回流闭环

8. 平台复制/导出包
9. 定时发布状态机和授权适配器
10. CSV / Excel 运营数据导入、诊断和再创作

### 阶段 D：多模态与商业运行

11. 视频上传、转写、镜头/字幕分析和改进 Brief

13. 全链路浏览器 E2E、三语检查、README 与发布证据更新

## 完成标准

- README 八个方向均有真实页面、API、数据模型、权限和测试证据。
- 未配置外部密钥时，下载仓库后仍能完成一条端到端演示链路。
- 配置合法 Provider 后无需改代码即可切换真实模型或平台连接。
- 至少覆盖 3 个农产品、3 个平台、3 种语言的内容生成与质量回归。
- Ruff、Mypy、Pytest、仓库审计、迁移测试、浏览器 E2E、Docker/Windows 包和 CI 全部通过。
- README 删除已经完成的“接下来建设”条目，改为实际能力、配置方法和尚需外部授权的连接状态。
