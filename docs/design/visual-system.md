# 禾语 AI 视觉体系与信息架构提案

> 本文只定义产品视觉、信息架构与展示规则，不直接修改正式首页。依据当前
> `README.md`、`docs/product.md` 和 `apps/web` 已实现界面整理；仓库当前没有
> `docs/roadmap.md`。

## 1. 用户首先需要看见什么

首页不是功能表单的集合，而是让访问者在较短时间内完成四个判断：

1. **这是做什么的**：把经过审核的农产品知识转化为可追溯的营销内容。
2. **为什么值得信任**：事实先审核、生成留来源、内容有版本、发布后有证据闭环。
3. **现在能完成什么**：品牌和产品建档、知识治理、短视频与直播内容生成、审核发布、
   运营快照和人工视频诊断。
4. **下一步怎么开始**：本地零成本进入工作台，或查看完整能力地图。

信息优先级：

| 优先级 | 用户问题 | 首屏表达方式 |
| --- | --- | --- |
| P0 | 平台能给我带来什么结果？ | 一句价值主张 + 一条端到端内容链 |
| P0 | 它是不是普通 AI 文案工具？ | 可追溯、可审核、版本不可覆盖的差异点 |
| P1 | 哪些功能现在能用？ | 带明确状态的能力地图 |
| P1 | 适合谁？ | 合作社、农业品牌、内容团队的角色场景 |
| P2 | 如何运行、是否收费？ | 本地优先、SQLite、可选 Docker/PostgreSQL |
| P2 | 未来会发展成什么？ | 独立“规划中”区域，不混入已上线入口 |

工作台则按任务排序，不再承担品牌宣传：

1. 待处理事项与下一步动作；
2. 品牌、产品和可信知识是否完整；
3. 正在创作及待审核内容；
4. 已发布内容和最新运营证据；
5. 团队权限、审计和系统状态。

## 2. 首页完整分区

### A. 顶部导航

- Logo 与“禾语 AI”
- 能力地图、工作流程、可信机制、开源运行
- 主按钮“进入工作台”
- 导航背景随滚动从透明变为半透明玻璃层，但不使用厚重阴影

### B. Hero：一句结果 + 一条活的内容链

主标题建议：

> 让每一份真实价值，成为可持续生长的内容。

说明：

> 从品牌与产地事实出发，建立经过审核的可信知识；再生成短视频、直播和社交内容，
> 保留每次引用、修改与发布证据。

主动作“进入工作台”，次动作“查看能力地图”。右侧不是缩小版后台，而是一条缓慢
运行的内容链：资料进入、知识审核、AI 创作、人工定稿、发布改进。节点只展示状态，
不放表单。

### C. 信任证据带

四项即可，不堆数字：

- 多租户权限隔离
- 审核知识才可生成
- 内容与来源可追溯
- 本地零成本可启动

### D. 能力地图

用六个领域容器完整展示平台，而非四张同质卡片：

1. **智能资产**：品牌档案、产品档案、表达边界；
2. **可信知识**：文本导入、审核、修订链、上下文证据；
3. **AI 创作**：短视频、直播、评论回复、社交文案、标题封面；
4. **质量治理**：内容版本、提交审核、批准/驳回、生成 provenance；
5. **发布运营**：发布登记、原始指标快照、发布详情；
6. **诊断改进**：结构化人工诊断、改进 Brief、后继草稿。

每个领域显示状态、适用角色和一项代表性输出。已上线卡片可进入对应 URL，规划卡片
不可伪装成按钮。

### E. 端到端工作流程

使用横向轨道或滚动叙事：

`建档 → 审核知识 → 创建 Brief → AI 生成 → 人工版本 → 审核 → 发布 → 诊断 → 后继草稿`

强调两个不可逆边界：审核历史不可覆盖、已发布版本不会被诊断自动改写。

### F. 角色场景

按任务而非职位宣传：

- 负责人：组织、成员和边界；
- 产品经理：产品事实和来源；
- 内容创作者：Brief、生成和人工修改；
- 审核者：知识与内容决策；
- 运营人员：发布数据、诊断和改进。

### G. 开源与运行

清楚展示两个启动路径：

- 零成本：Windows + SQLite + deterministic provider；
- 生产式：Docker Compose + PostgreSQL + 可替换 AI Provider。

这里不声称 ChatGPT 订阅等于 API 额度，也不暗示本地已有真实大模型。

### H. 规划中

放在已上线能力之后，视觉降权：

- PDF、DOCX、PPTX 解析；
- 自动媒体提取辅助诊断；
- 全文检索或 Embedding 检索；
- PostgreSQL RLS 防御增强；
- 独立 Windows 运行时安装包。

不进入路线图：VR、医疗、数字人直播、联邦学习、区块链、无真实证据的爆款预测。

### I. 最终 CTA 与页脚

CTA：“先在本机建立第一份可信内容资产”。页脚保留 GitHub、文档、许可、隐私边界和
版本状态。

## 3. 已上线与规划中的区分

状态必须同时依靠**文字、图形和交互**，不能只靠颜色：

| 状态 | 标识 | 卡片表现 | 交互 |
| --- | --- | --- | --- |
| 已上线 | 实心圆 + `已上线` | 清晰、正常对比 | 可进入真实工作台 URL |
| Beta | 半实心圆 + `Beta` | 正常对比，附边界说明 | 可进入，但注明人工/实验限制 |
| 规划中 | 空心圆 + `规划中` | 低一级对比，无悬浮抬升 | 不提供假入口，只展开说明 |
| 明确排除 | 不在产品内 | 不放在能力地图 | 仅在范围说明中出现 |

当前能力状态建议：

- 已上线：多租户/RBAC、品牌产品维护、文本知识导入与审核、知识修订链、内容 Brief、
  八类内容生成、内容版本审核、审计、发布记录、原始运营快照。
- Beta：deterministic 本地 AI、lexical-v1 有界知识选择、人工结构化视频诊断、
  诊断到改进 Brief 和后继草稿。
- 规划中：文档解析、真实模型 Provider 配置界面、自动媒体提取、更强检索、
  独立 Windows 包。

首页不得用“智能预测”“自动爆款”“自动诊断完成”等文案。

## 4. 动效规则

### 环境动效

- 日光和极光使用 18–36 秒循环，位移不超过容器尺寸的 12%。
- 使用 2–3 个大尺度层，不同时堆叠大量小光点。
- 极光运动方向保持一致的主流向，避免无规则交叉。
- 动效服务于层级：Hero 最强，能力区减弱，工作台只保留静态环境色。

### 信息动效

- 内容链的节点每 2.4–3.2 秒依次高亮，不快速闪烁。
- 路径流光只沿一条线移动，完成一轮后停顿。
- 卡片进入采用 12–20px 轻位移和透明度，持续 420–650ms。
- hover 只使用 1–2px 位移、边框和局部光，不整卡大幅缩放。

### 无障碍与性能

- 完整支持 `prefers-reduced-motion`，关闭位移、循环和自动滚动。
- 动画只改变 `transform` 和 `opacity`，模糊背景尽量静态。
- 文本、表单和关键状态不依赖动画才能读取。
- 移动端减少极光层数并停用跟随鼠标效果。

## 5. 字体、字号与行距

### 字体

- 中文/正文：`Inter, "Segoe UI", "Microsoft YaHei", sans-serif`
- 展示标题：同一字体族，以字重和字距区分，不额外引入装饰字体
- 数据/状态：`"Cascadia Code", Consolas, monospace`

### 响应式字号

| 层级 | 桌面 | 移动 | 行高 | 字距 |
| --- | --- | --- | --- | --- |
| Hero H1 | `clamp(52px, 6vw, 88px)` | 42–56px | 1.06–1.12 | -0.045em |
| 分区 H2 | `clamp(36px, 4vw, 58px)` | 32–42px | 1.14–1.22 | -0.035em |
| 卡片 H3 | 22–30px | 20–26px | 1.25–1.35 | -0.02em |
| 工作台页标题 | 24–30px | 22–26px | 1.25 | -0.025em |
| 正文重点 | 17–19px | 16–18px | 1.7–1.85 | 0 |
| 正文 | 14–16px | 14–16px | 1.65–1.8 | 0 |
| 辅助文字 | 12–13px | 12–13px | 1.55–1.7 | 0.01em |
| 标签/状态 | 11–12px | 11–12px | 1.4–1.55 | 0.06–0.1em |

规则：

- 不再出现 9px 正文；9–10px 只允许用于非关键装饰编号。
- 中文大标题每行建议 8–16 字，最多三行；正文每行约 30–42 个中文字符。
- 标题与说明间距 20–28px，说明与按钮 28–36px，分区垂直间距 120–180px。
- 工作台优先密度和可扫读性，不复制首页超大标题。

## 6. 两套可视化方向

### 方向 A：日光极光 / Sunlit Aurora（推荐深化）

- 主背景：晨光灰白 `#F4F8F4`
- 深墨绿：`#11362B`
- 薄荷绿：`#54C99A`
- 日光黄：`#F3D66B`
- 少量天空青：`#8ED8D1`
- 特征：明亮、可信、有生命力；大面积环境渐变承载氛围，重要内容使用纯色实体块。
- 适合：品牌首页、开源演示、农业内容平台长期主视觉。
- 原型：`outputs/design-concepts/sunlit-aurora.html`

### 方向 B：晨雾珊瑚 / Field Signal

- 主背景：冷灰蓝 `#E8EEF0`
- 深蓝绿：`#17383B`
- 珊瑚：`#EF765F`
- 信号绿：`#38B889`
- 柔和天空蓝：`#85BFD1`
- 特征：更像内容运营和生产系统，珊瑚只用于关键行动、运行节点和状态信号。
- 适合：强调商业运营、数据和内容生产效率的版本。
- 原型：`outputs/design-concepts/field-signal.html`

两套方向都避免“全屏纯白 + 同质卡片”的 SaaS 模板感，也不使用深黑背景。正式实现建议以
方向 A 为品牌首页，以方向 B 的信息密度方法优化工作台，而不是让两套主题同时存在。
