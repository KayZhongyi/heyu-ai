# 开源能力适配说明

禾语 AI 会优先复用成熟、许可证兼容的开源能力，但不会把第三方 Skill 或项目直接复制进业务代码。
接入前必须确认许可证、依赖体量、安全边界、可测试性和长期维护成本，再通过禾语 AI 的适配层连接到
权限、知识库、审核、版本和审计流程。

## 当前采用

| 能力 | 上游项目 | 许可证 | 禾语 AI 中的用途 |
| --- | --- | --- | --- |
| PDF 文字抽取 | `py-pdf/pypdf` | BSD-3-Clause | 在内存中解析 PDF，返回页级文本与警告，不永久保存原文件 |
| PPTX 读取与生成 | `scanny/python-pptx` | MIT | 抽取幻灯片文字，并生成原生可编辑的营销方案 PPTX |
| 文件上传 | `Kludex/python-multipart` | Apache-2.0 | 处理 FastAPI 受限文件上传 |

当前适配不依赖 Docker、Ollama、LibreOffice、外部 OCR 或付费模型 API。

## 已调研、暂未默认接入

| 项目 | 许可证 | 结论 |
| --- | --- | --- |
| `microsoft/markitdown` | MIT | 支持多种 Office 与文档格式，适合后续扩展 DOCX、XLSX 等导入；当前 PDF / PPTX MVP 使用更小的直接依赖，避免无必要增加安装体积 |
| `Unstructured-IO/unstructured` | Apache-2.0 | 文档分块能力完整，但依赖和运行成本较高；待批量资料处理、复杂版面和 OCR 需求明确后再评估 |
| Anthropic 官方 PPTX Skill | 专有许可 | 许可证不允许复制或创建衍生作品，因此不纳入仓库；仅参考公开的产品交互思路，不复用其代码或提示词 |

## 新能力接入规则

1. 先记录来源仓库、固定版本和许可证。
2. 第三方库只负责底层解析或生成，不绕过禾语 AI 的租户权限与审核流程。
3. 上传内容默认在本地处理；任何外部服务必须由负责人明确授权。
4. 为正常文件、损坏文件、超限文件、跨租户访问和可重新打开的导出文件补自动化测试。
5. 功能进入平台页面和 API 后，才算完成；孤立脚本或无法审计的演示不计入当前能力。
6. 进入正式发布前锁定依赖版本，生成 SBOM 与 `THIRD_PARTY_NOTICES`，并复查许可证、已知漏洞和上游维护状态。
